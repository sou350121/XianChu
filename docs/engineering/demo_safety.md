# Demo Safety（现场安全 runbook）

## 目标
确保演示中：
- **人不受伤**
- **琴不受损**
- **设备可快速复位**
- 异常时可快速止损（急停/撤离/安全姿态）

SSOT：`../../contracts/safety/limits.md`

## 1) 场地与人员（演出前必须明确）

- **围栏/隔离线**：观众最近距离、是否需要围栏
- **安全员**：位置、职责（仅安全员可复位/解除 e-stop）
- **急停按钮**：物理急停位置（机器人本体 + 外置按钮）
- **通道**：电源/网线/线缆走线，防绊倒、防拉拽

## 2) 演示前检查（Checklist）

- [ ] 软/硬急停可用（按下后系统进入安全态；解除需要明确流程）
- [ ] 已加载对应 demo profile 与 `contracts/safety/limits.md` 的阈值配置
- [ ] `reset_safe_pose` 可用（可在不碰撞的情况下撤离）
- [ ] 琴与弓握持检查：无松动、无干涉、线缆不勾挂
- [ ] 触觉/关节状态/音频通道在线（心跳/超时门禁生效）
- [ ] 触觉失联策略确认：失联是否会触发 recovery 或 e-stop（尤其 `violin_grip`）
- [ ] 低速试运行：轻触弦/轻运弓，确认接触事件与音频响应合理

## 3) 演示中异常处理（强制动作）

- **碰撞/夹伤风险**：立刻急停 → 撤离 → 复位流程
- **掉落风险（持琴滑移不可控）**：优先保琴
  - 进入握持增强恢复；若不可恢复 → e-stop
- **触觉/状态失联超时**：视为高危
  - 直接进入安全状态（recovery 或 e-stop，按 `limits.md`）
- **啸叫/强噪声持续**（不危及安全但影响观感）：降弓压/降速/短撤离后重接触

## 4) 演示后

- [ ] 导出日志/rosbag（登记到 `../../data_registry/external_artifacts.md`）
- [ ] 记录失败路径与失败模式（写入评测汇总；必要时补 ADR/复盘笔记）
- [ ] 检查弓毛/琴弦/指板是否有异常磨损（如有，立刻降级 demo 强度）
